## 1. Introduction

### 1.1 연구 배경

스마트팜 관련 연구를 하다 보면 실제로 가장 귀찮은 게 작물 상태 확인이다. 드론이나 카메라로 전체를 찍는 방법도 있지만, 해충은 가까이서 봐야 알 수 있고 수확 적기도 직접 확인해야 정확하다. 자율 이동 로봇이 직접 돌아다니면서 필드를 순찰하고 조치까지 취하는 시스템을 강화학습으로 구현해보기로 했다.

### 1.2 강화학습 적용 근거

본 문제는 다음 세 가지 이유로 강화학습이 적합하다.

**순차적 의사결정**: 현재의 이동 선택이 이후 도달 가능한 영역과 잔여 작업량을 바꾼다. 매 스텝의 행동이 미래 보상에 영향을 주므로 단순 분류·회귀가 아닌 누적 보상 최대화 문제다.

**부분 관측(POMDP)**: 로봇은 예찰(Scout) 행동을 하기 전에는 작물의 실제 상태를 알 수 없다. "정보를 얻는 행동(예찰)"과 "정보를 활용하는 행동(조치)"이 분리되어 있어, 에이전트(또는 계층적 구조에서는 상위 정책)가 탐색과 활용의 순서를 스스로 결정해야 한다.

**경로 최적화의 조합론적 복잡성**: 5개 레인 방문 순서만 해도 최소 120가지(5!) 이상이며, 매 에피소드마다 작물 상태가 랜덤하게 바뀌므로 고정 규칙으로는 최적 대응이 불가능하다. 개별 작업은 규칙화할 수 있어도 이동 경로는 보상 신호로 학습하는 것이 적합하다.

### 1.3 연구 질문 및 검증 설계

본 연구는 위 세 특성을 만족하는 커스텀 환경을 구현하고, 다음 네 가지 연구 질문에 단계적으로 답한다.

| 단계 | 연구 질문 | 검증 대상 |
|------|----------|----------|
| Step 1 | 이산 격자 위에서 단일 정책(Flat RL)으로 본 문제를 학습할 수 있는가? | MaskablePPO + Action Masking |
| Step 2 | "전역 계획"과 "국소 실행"을 계층 구조로 분리하면 효율이 향상되는가? | Hierarchical PPO |
| Step 3 | 상위 정책의 관측에 거리 정보를 추가하면 최적 정책(Greedy)을 자동으로 발견하는가? | DQN + 거리 인식 obs |

## 2. Materials and Methods

본 절은 §1.3의 네 연구 질문을 검증하기 위해 설계한 환경과 알고리즘을 기술한다. 결과·해석은 다루지 않는다.

### 2.1 공통 환경 설계

#### 2.1.1 농경지 구조: 2열 재배단

현실 온실의 수직재배 구조를 모델링한 **2열 재배단(two-column bed)** 격자 맵(Fig. 1)을 사용한다. 토마토·파프리카와 같은 작물은 줄기 양쪽으로 열매가 분포하므로, 로봇이 한쪽 레인에서 접근하면 안쪽 열만 스캔 가능하고 바깥쪽 열은 반대편 레인에서만 작업 가능하다. 이 **양면 접근(double-sided access) 제약**을 격자 상에서 표현하기 위해 단일 작물 줄을 안쪽·바깥쪽 두 열로 추상화하였다.

![Fig. 1. 최종 농경지 맵 구조](../assets/figures/map_structure.png)

```
n_beds=4, field_height=8 → H=12, W=15

r0:  W W W W W W W W W W W W W W W
r1:  W P P P P P P P P P P P P P W   ← 상단 헤드랜드
r2:  W P C C P C C P C C P C C P W   ← 필드 행 (r2~r9)
r10: W P P P P P P P P P P P P P W   ← 하단 헤드랜드
r11: W W W W W W W W W W W W W W W

P=주행 레인, C=작물 셀, W=벽 / 레인 col: 1,4,7,10,13 (5개)
```

상단·하단 헤드랜드는 레인 간 이동을 매개하는 유일한 통로다. 레인 전환에는 실질적 이동 비용이 발생한다.

| 항목 | 값 |
|------|-----|
| 맵 크기 | 12 × 15 |
| 작물 셀 수 | 64개 |
| 주행 레인 수 | 5개 |
| 에이전트 시작 | (1, 1) — 상단 헤드랜드 |

#### 2.1.2 작물 상태 동역학 (POMDP)

각 작물은 에피소드 시작 시 아래 분포에 따라 무작위로 초기화되며, 실제 상태는 에이전트에게 **은닉**된다. 환경은 내부적으로 `_true_states`(은닉)와 `crop_states`(공개)를 분리해 관리한다. 예찰(Scout) 액션을 수행한 작물만 `crop_states`에 실제 상태가 공개되며, 이로 인해 환경은 POMDP 성격을 띤다.

| 초기 상태 | 확률 | 종결 조건 |
|----------|------|----------|
| 정상 (1) | 60% | 예찰만으로 확인 완료 |
| 수확 필요 (2) | 25% | 수확 액션으로 수확 완료(4) |
| 방제 필요 (3) | 15% | 방제 액션으로 방제 완료(5) |

모든 작물이 종결 상태 {1, 4, 5} 중 하나에 도달하면 에피소드가 성공 종료된다.

#### 2.1.3 보상 함수 (이산 환경 기본)

| 이벤트 | 보상 |
|--------|------|
| 매 스텝 | −0.1 |
| 충돌 시도 | −2.0 |
| 신규 셀 예찰 | +1.0 |
| 정상 확인 | +0.5 |
| 수확 성공 | +10.0 |
| 방제 성공 | +8.0 |
| 전체 완료 | +20.0 |

각 Step에서 변경된 보상 항목은 해당 절에서 별도 기술한다.

### 2.2 Step 1 — 이산 Flat RL

#### 목적
이산 격자에서 단일 정책으로 §1.3의 문제를 학습 가능한지 검증한다.

#### 환경 — FarmEnv

**관측**: `Box(0, 1, shape=(720,))` — 4채널 × 12 × 15

| 채널 | 내용 | 정규화 |
|------|------|--------|
| ch0 | 맵 구조 (0=통로, 1=작물, 2=벽) | ÷2 |
| ch1 | 에이전트 위치 one-hot | 0/1 |
| ch2 | 예찰 완료 마스크 | 0/1 |
| ch3 | 예찰 결과 (0=미지~5=방제완료) | ÷5 |

ch3는 §2.1.2의 POMDP 특성을 관측 벡터에 직접 반영한다. 예찰 전 작물은 0(미지)으로만 표현되므로 탐색 유인이 관측 구조에서 발생한다.

**행동**: `Discrete(7)` + Action Masking

| 인덱스 | 액션 | 마스킹 조건 |
|--------|------|------------|
| 0~3 | 이동 (상/하/좌/우) | 목적지가 작물/벽 |
| 4 | 예찰 (Scout) | 인접 미예찰 셀 없음 |
| 5 | 수확 | 인접 수확대기 셀 없음 |
| 6 | 방제 | 인접 방제대기 셀 없음 |

Action Masking은 무효 액션의 로짓을 −∞로 치환해 softmax 후 선택 확률을 0으로 만든다.

**종료**: 전체 작물 처리 완료 OR max_steps = 540

#### 알고리즘 — MaskablePPO

이산 행동 공간 + 동적 Action Masking을 동시에 지원하는 sb3-contrib 구현.

| 하이퍼파라미터 | 값 |
|--|--|
| n_envs (DummyVecEnv) | 16 |
| total_timesteps | 500,000 |
| learning_rate | 3×10⁻⁴ |
| n_steps | 512 |
| batch_size | 256 |

### 2.3 Step 2/3 — 이산 계층형 RL

#### 목적
- Step 2: 전역 계획(레인 선택)과 국소 실행(레인 처리)을 계층 분리하여 효율이 향상되는지 검증.
- Step 3: 거리 관측·목표 도달 보상·DQN을 포함한 개선 구성이 안정적인 정책 학습에 기여하는지 검증.

#### 환경 — LaneExecutorEnv + HighLevelFarmEnv

**Low-level (LaneExecutorEnv)**: FarmEnv를 확장. ch4(목표 레인 마스크) 채널 추가 → 5채널 × 900-dim. 상위 정책이 지정한 목표 레인이 모두 처리되면 에피소드 종료(max_steps = 180/레인). 레인 완료 시 추가 보상 +10.0 (`REWARD_LANE_COMPLETE`).

**High-level (HighLevelFarmEnv)**: Low-level 정책을 호출하는 메타 환경. 한 step()마다 Low-level이 목표 레인 처리 종료까지 실행된다.

| 항목 | Step 2 (PPO HL) | Step 3 (DQN HL) |
|------|----------------|----------------|
| HL obs | 레인별 완료율(5) | 레인별 완료율(5) + **거리(5) = 10-dim** |
| HL action | Discrete 5 (레인 선택) | Discrete 5 |
| HL 알고리즘 | PPO | DQN |

Step 3의 거리 항목은 현재 에이전트 위치에서 각 레인 col까지의 정규화 거리다. §3에서는 거리 관측을 포함한 Step 3 최종 구성의 결과와 Greedy nearest-lane 규칙의 집계 성능을 비교한다.

#### 알고리즘

| | Low-level | High-level (Step 2) | High-level (Step 3) |
|--|--|--|--|
| 알고리즘 | MaskablePPO | PPO | DQN |
| total_timesteps | Step 2: 500,000 / Step 3: 700,000 | 200,000 | 300,000 |
| 비고 | Action Masking 유지 | On-policy | Off-policy, replay buffer |

### 2.4 Step 4 — 연속 환경 + 연속 제어

#### 목적
이산 격자 단순화를 넘어 연속 2D 좌표·연속 속도 제어로 정책이 확장 가능한지 검증한다. SAC·TD3·TQC 세 알고리즘의 적합성을 비교한다.

#### 환경 — ContinuousFarmEnv

| 항목 | 이산 환경 | 연속 환경 |
|------|----------|----------|
| 위치 | 격자 인덱스 | 실수 좌표 (x, y) |
| 이동 | 칸 단위 (4방향) | 속도 벡터 (vx, vy) |
| 예찰/조치 | 명시적 Scout/Harvest/Pest 액션 | 거리 임계값 이내 자동 |
| 충돌 | 목적지 셀 타입 | 기하학적 겹침 |

**관측**: `Box(-1, 1, shape=(28,))`

| 구성 | 차원 | 내용 |
|------|------|------|
| robot_pos + heading | 4 | 위치 (x, y) + 이동 방향 (cos, sin) |
| nav_flags | 4 | 4방향(N/S/E/W) 이동 가능 여부 |
| top-5 nearest crops | 20 | 가장 가까운 미처리 작물 5개의 (dx, dy, revealed, state) |

nav_flags는 로봇이 어느 방향이 막혀 있는지를 명시적으로 제공한다. 이 특징을 포함한 최종 구성에서 높은 성능이 측정되었으나, 독립 효과는 추가 통제 실험이 필요하다.

**행동**: `Box(-1, 1, shape=(2,))` — 속도 벡터 (vx, vy)

**보상**: 이산 대비 스케일 조정 (`R_STEP=−0.05`, `R_COLLISION=−1.0`, `R_SCOUT_NEW=+1.5`) + Potential-based shaping:

$$F(s, s') = \gamma\,\phi(s') - \phi(s), \quad \phi(s) = -\,\text{min\_dist}(s) \times 0.3$$

Ng et al.(1999)의 잠재 함수 차분 방식은 최적 정책 불변성을 보장하면서 희박 보상 환경의 학습 신호를 조밀하게 만들어 준다.

**종료**: 전체 작물 처리 완료 OR max_steps = 1200

`ContinuousFarmEnvCurriculum`은 단계별 스폰 기능을 지원하지만, 최종 학습 스크립트에서는 `curriculum_level = 2`로 고정하여 모든 에피소드를 헤드랜드에서 시작한다. 최종 성능 개선은 28차원 관측 단순화와 `nav_flags` 적용 조건에서 측정하였다.

#### 알고리즘 — SAC / TD3 / TQC 비교

세 알고리즘은 동일한 연속 환경 계열에서 비교하되, 최종 학습 스크립트의 학습 예산과 업데이트 설정은 알고리즘별로 다르다.

| | SAC | TD3 | TQC |
|--|--|--|--|
| 정책 형태 | Stochastic, 최대 엔트로피 | Deterministic + exploration noise | Stochastic, distributional Q |
| total_timesteps | 2,000,000 | 1,500,000 | 1,500,000 |
| batch_size | 256 | 4,096 | 4,096 |
| train_freq | 1 | 16 | 16 |

## 3. Results

### 3.1 Step별 종합 성능

표 1은 §1.3의 네 연구 질문에 해당하는 단계별 정량적 성능을 요약한다. 모든 수치는 동일한 평가 절차(각 단계 ≥ 20 에피소드, 결정론적 정책, 서로 다른 시드)로 측정되었다.

**표 1.** Step별 최종 성능 비교

| 단계 | 알고리즘 | 성공률 | 커버리지 | 평균 스텝 | 평균 레인 방문 |
|------|----------|--------|---------|---------|--------------|
| Step 1 | MaskablePPO (Flat) | 96% | 99.8% | 147 ± 80 | — (Flat이므로 측정 불가) |
| Step 2 | Hierarchical PPO | 98% | 99.8% | 198 ± 349 | 2.3 |
| **Step 3** | **DQN + 거리 인식 HL** | **100%** | **100%** | **141 ± 3** | **2.0** |
| Step 4 | SAC + 단순화 관측 | 96.7% | 99.9% | 157 ± 194 | — (Flat이므로 측정 불가) |

(Fig. 2 — `assets/figures/results_step3_comparison.png` 참조: Step 1~3의 보상·커버리지·스텝 분포 히스토그램)

표 1과 Fig. 2에서 두 가지 패턴이 관찰된다. 첫째, **Step 3에서 평균 스텝 표준편차가 ±3으로 압도적으로 낮다** — 다른 단계는 ±80~349 수준이다. 둘째, Step 1→2로 가는 변화(96%→98%)보다 Step 2→3 변화(98%→100%)가 작아 보이지만, 평균 스텝은 198→141로 약 29% 감소하여 정책 품질의 개선이 더 크다.

### 3.2 Step 1 — Flat RL의 학습·정책 특성

Step 1의 최종 학습 스크립트는 총 500,000 스텝을 사용한다. 최종 평가에서 96% 성공률과 99.8% 평균 커버리지를 기록했으며, 이는 대부분의 에피소드에서 거의 모든 작물을 처리하는 정책이 학습되었음을 보여준다. 다만 저장소에는 원시 학습 로그가 포함되어 있지 않아 세부 수렴 시점을 현재 상태에서 독립적으로 재검증할 수는 없다.

대표 정책 시연은 저장소의 `assets/demos/step1.gif`에서 확인할 수 있다.

GIF에서 두드러지는 행동 패턴은 **Boustrophedon(지그재그) 순회**다. 에이전트는 한 레인을 끝까지 내려간 뒤 헤드랜드를 거쳐 인접 레인으로 전환하고 다시 반대 방향으로 내려가는 패턴을 일관되게 보였다. 이 패턴은 환경 설계나 보상 함수에 명시적으로 인코딩되지 않았으며 오로지 "스텝당 −0.1" 패널티와 작업 보상의 균형에서 자발적으로 출현하였다.

성공률 4%(2/50)의 실패는 모두 max_steps=540 초과 종료였으며, 해당 에피소드들도 평균 85% 이상의 커버리지에 도달한 상태로 종료되었다 — 즉 정책이 작업을 포기한 것이 아니라 시간이 부족했던 것이다.

### 3.3 Step 2·3 — 계층 분리와 Step 3 개선 구성의 결과

### 3.6 보조 실험 — Action Masking의 효과

같은 이산 환경에서 마스킹을 사용하는 MaskablePPO와 마스킹을 사용하지 않는 RecurrentPPO의 보조 비교를 수행하였다. 두 방법은 마스킹 여부뿐 아니라 정책 구조도 다르므로 마스킹만의 독립 효과를 측정한 실험은 아니다.

**표 5.** Action Masking이 학습에 미치는 영향

| 알고리즘 | 마스킹 | 학습 스텝 | 성공률 | 커버리지 |
|---------|--------|---------|--------|---------|
| RecurrentPPO (LSTM) | 미지원 | 500,000 | 0% | 0.9% |
| MaskablePPO | 지원 | 500,000 | 96% | 99.8% |

RecurrentPPO는 충돌 패널티(−2.0)로 무효 액션을 사후 억제하는 학습 경로를 따라야 한다. 50만 스텝 학습 후 커버리지가 1% 미만에 머문 반면, MaskablePPO 최종 구성은 높은 성공률을 기록했다. 이 결과는 본 보조 비교에서 마스킹을 포함한 구성이 효과적이었음을 보여주지만, 마스킹만의 효과를 확정하려면 동일 정책 구조의 통제 실험이 필요하다.

## 4. Discussion

### 4.1 §1.3 연구 질문에 대한 답

**Q1: 이산 격자에서 Flat RL로 학습 가능한가?**

가능하다. Step 1에서 MaskablePPO는 96% 성공·99.8% 커버리지를 달성하였다. 더 흥미로운 것은 학습된 행동 패턴이다. 환경에는 "지그재그로 순회하라"는 사전 지식이 어떤 형태로도 인코딩되어 있지 않으나, 보상 신호(스텝 패널티 + 작업 보상 + 완료 보너스)만으로 에이전트는 Boustrophedon 패턴을 자발적으로 발견하였다. 이는 RL이 단순한 누적 보상 최대화를 통해 도메인 휴리스틱에 해당하는 거동을 스스로 학습할 수 있음을 보여준다.

다만 §3.1에서 보듯 평균 스텝 표준편차가 ±80으로 변동이 크고, 4%의 시간 초과 실패가 존재한다. 이는 단일 정책이 "어느 레인을 갈지"와 "지금 어떻게 움직일지"라는 두 결정 수준을 동시에 학습하면서 발생하는 비효율로 해석할 수 있고, 이것이 Q2의 동기가 된다.

**Q2: 계층 분리로 효율이 향상되는가?**

부분적으로만 그렇다. Step 2의 성공률은 96%→98%로 소폭 개선되었으나 평균 스텝은 오히려 198로 증가하였고 표준편차도 ±349로 매우 컸다. 즉 **분리 자체만으로는 일관된 효율 향상이 보장되지 않았다**. 분리된 상위 정책이 의미 있는 결정을 내리려면 적절한 정보가 관측에 들어와야 한다는 사실이 Step 3에서 드러난다.

**Q3: 거리 정보를 포함한 Step 3 구성이 안정적인 정책 학습에 기여하는가?**

Step 3 최종 구성은 성공률 100%, 평균 스텝 141 ± 3으로 안정적인 결과를 보였다. Greedy 규칙과의 비교에서도 평가한 집계 지표가 동일했다. 다만 Step 3에는 거리 관측 외에도 하위 보상·학습량, 상위 알고리즘·학습량 변경이 함께 포함되므로 거리 정보만의 인과 효과로 해석할 수 없다.

이 결과의 함의는 두 가지다. 첫째, **올바른 관측이 주어지면 RL은 단순 가치 기반 최적 휴리스틱을 자율적으로 발견할 수 있다** — 사람이 그 휴리스틱을 명시적으로 설계해 넣을 필요가 없다. 둘째, 거리 정보처럼 결정에 직접 관련된 특징을 관측에 포함시키는 것이, 같은 양의 정보를 보상으로 우회 표현하는 것보다 훨씬 효율적이다. 후자는 §4.5의 핵심 발견과 직접 연결된다.

**Q4: 연속 환경 확장 가능한가? 어떤 알고리즘이 적합한가?**

연속 환경에서도 96.7% 성공률·99.9% 커버리지의 정책 학습이 가능했다. 각 최종 스크립트 구성에서는 SAC가 가장 좋은 결과를 보였으며, 최대 엔트로피 탐색이 기여했을 가능성이 있다. 다만 알고리즘별 학습 예산과 업데이트 설정이 달라 이 결과만으로 SAC의 알고리즘적 우위를 직접 입증하지는 않는다.

TD3와 TQC의 낮은 성능에는 탐색 부족이 영향을 주었을 가능성이 있다. SAC의 자동 엔트로피 조절은 탐색을 유지하는 데 유리할 수 있으나, 현재 비교는 학습 설정도 서로 달라 이 요인을 원인으로 확정할 수 없다.

### 4.2 핵심 관찰: 관측 공간 설계의 중요성

본 개발 과정에서 가장 일관되게 관찰된 경향은 **결정에 직접 관련된 관측을 제공한 구성이 높은 성능과 연관되었다**는 점이다. 다만 아래 변경들은 완전히 독립적으로 통제된 실험이 아니므로 인과 효과의 크기를 확정하지는 않는다.

**표 6.** 관측 보강 vs 보상 강화의 효과 비교

| 변경의 유형 | 구체적 변경 | 성공률 변화 | 효과 방향 |
|------------|------------|-----------|---------|
| 보상 강화 | 스텝 패널티 −0.1 → −0.3 | 96% → 80% | 악화 |
| 구성 변경 | 거리 관측·DQN·학습 설정 변경 | ≤90% → 100% | 향상 |
| 관측 보강 | 연속 환경에 nav_flags 추가 | 6% → 56% | 대폭 향상 |

보상 강화는 일견 직관적인 개입이지만 학습 신호의 모양을 바꿔 의도치 않은 부작용을 일으킬 수 있다. 본 실험에서 스텝 패널티를 3배 강화하자 에이전트는 "빠르게 끝내는" 정책 대신 "위험을 회피하기 위해 아예 진입하지 않는" 정책으로 수렴하였다 — 보상 함수가 의도와 다른 최적해를 만들어 낸 것이다.

거리 정보는 상위 정책에 레인 선택과 직접 관련된 특징을 제공하고, nav_flags는 연속 정책에 이동 제약 정보를 제공한다. 이 특징들을 포함한 구성에서 성능이 개선되었지만 다른 설정도 함께 변경되었으므로, 개별 특징의 기여도는 추가 ablation으로 검증해야 한다.

이 관찰은 보상 공학에 집중하던 전통적인 RL 디자인 관행과 비교하여, 본 태스크 같은 **부분 관측·조합론적 문제에서는 관측 설계가 더 효과적인 개입 지점**임을 시사한다.

### 4.3 Action Masking의 필수성

§3.6의 보조 비교에서는 마스킹을 사용하는 MaskablePPO 구성이 RecurrentPPO 구성보다 높은 성능을 보였다. 본 태스크의 행동 공간은 7가지지만 매 스텝 유효 행동 집합이 제한적이므로, 마스킹이 무효 행동 표본을 줄이는 데 유리하다는 해석은 타당하다. 다만 두 정책 구조가 달라 마스킹의 독립 효과를 확정하려면 추가 통제 실험이 필요하다.

본 결과는 동적 제약이 강한 환경에서 마스킹이 단순한 효율 향상 기법이 아니라 학습 가능성을 좌우하는 구조적 요소임을 시사한다.

### 4.4 RL과 휴리스틱의 동등성: 의미와 함의

§3.4에서 평가한 집계 지표의 일치는 양면의 함의를 갖는다. 한편으로 본 태스크의 상위 의사결정 문제가 충분히 정형화되어 있어, 결정론적 휴리스틱으로도 높은 성능을 표현 가능함을 의미한다. 따라서 본 사례에서는 **상위 정책에 굳이 RL을 사용할 필요가 없을 수도 있다**.

그러나 다른 한편으로, 평가 범위 안에서는 RL이 **수동 휴리스틱과 같은 집계 성능에 도달**할 수 있음을 보여준다. 동역학이 더 복잡하거나 휴리스틱을 명시하기 어려운 변형 문제에서는 RL의 자동 발견 능력이 더 큰 가치를 가질 수 있다.

### 4.5 한계 및 향후 연구

본 연구는 다음과 같은 한계를 가진다.

첫째, **연속 환경의 계층형 RL은 1M 스텝 설정에서 낮은 커버리지에 머물렀다**(§5.4). 하위 환경이 "지정 레인까지 이동 후 그 레인만 처리하고 종료"하는 goal-conditioned 태스크라는 점이 난이도에 영향을 주었을 수 있다. Flat SAC 최종 구성은 2M 스텝에서 96.7% 성공률을 기록한 반면, goal-conditioned SAC 구성은 1M 스텝에서 41.7% 커버리지를 기록했다. 학습량과 태스크가 함께 다르므로 직접 비교에는 한계가 있으며, 향후 더 긴 학습 또는 HER 같은 전용 기법을 검토할 수 있다.

둘째, **본 환경은 단일 로봇·정적 작물 분포를 가정**한다. 실제 농경지는 작물 성장 주기에 따라 상태가 시간적으로 변화하고, 복수 로봇이 동시에 작업할 수 있다. 향후 MARL(예: MAPPO)로 분할 예찰을 학습하고, 동적 작물 상태를 도입하여 정책의 강건성을 검증할 수 있다.

셋째, **본 연구는 시뮬레이션에 한정된다**. ROS2 아키텍처(§1.4)는 배포를 염두에 둔 설계이지만 실제 로봇 검증은 수행하지 않았다. 학습된 SAC 정책의 속도 벡터 출력을 `cmd_vel` 토픽으로 연결해 실 차량에서 검증하는 작업이 필요하다.

## 5. R&D 시행착오

본 절은 §2~§4에서 다루지 않은, 최종 결과에 이르기까지의 실패와 우회 과정을 기록한다. 각 시도는 "시도 → 가설 → 결과 → 교훈" 구조로 정리한다.

### 5.1 환경 설계: 1열 → 2열 재배단

**시도**: 초기 환경은 단일 작물 열에 좌우 통로가 인접한 1열 재배단(작물 24셀, n_lanes=3, field_height=6)이었다.

**가설**: 1열 구조는 하나의 레인에서 좌우 작물을 모두 스캔 가능하므로 학습은 더 쉬울 것이고, 핵심 검증(POMDP·계층 분리)은 그대로 가능할 것이다.

**결과**: 개발 기록상 학습은 잘 되었으나(96% 성공) 평균 레인 방문이 약 1.0회로 사실상 "한 레인 통과로 끝"이라는 자명한 정책에 수렴해, 5개 레인 방문 순서 최적화라는 핵심 문제가 발생하지 않았다. 초기 버전의 원시 로그와 코드는 현재 저장소에 보존되어 있지 않다.

**교훈**: 현실 농경지의 양면 접근 제약을 모델링하지 않으면 RL이 풀어야 할 진짜 문제(레인 순서 결정)가 사라진다. 2열 재배단(64셀, n_beds=4)으로 변경하여 모든 레인을 방문해야만 전체 커버리지가 달성되도록 만들었다.

### 5.2 보상 스케일링 실험 — Step 패널티 강화

**시도**: 학습된 Step 3 정책의 평균 스텝(141)을 더 줄이기 위해 매 스텝 패널티를 −0.1 → −0.3으로 3배 강화하였다.

**가설**: 시간 비용이 커지면 에이전트가 더 적극적으로 효율을 추구하여 스텝 수가 감소할 것이다.

**결과**: 개발 기록상 성공률이 96%에서 80%로 감소했고 평균 스텝도 줄지 않았다. 원시 로그가 현재 저장소에 남아 있지 않으므로 통제 실험으로 재검증된 결과는 아니다.

**교훈**: 보상 강화는 의도와 다른 최적해를 만들 수 있다. 정책의 행동을 바꾸려면 보상 스케일을 키우는 것보다 관측을 보강해 결정의 근거를 제공하는 편이 효과적이다. 이 관찰이 §4.5의 핵심 발견 형성에 직접 기여하였다.

### 5.3 SAC 연속 환경의 4번 반복 개선

연속 환경의 학습은 한 번에 성공하지 않았다. 다음은 개발 과정에서 기록한 네 구성과 결과다. 여러 변경이 단계별로 함께 적용되었고 원시 로그가 현재 저장소에 모두 보존되어 있지 않으므로, 개별 변경의 인과 효과를 분리한 ablation으로 해석하지 않는다.

**v1 — 기본 SAC, 124-dim 관측 (커버리지 0%)**

*시도*: 30개 작물 전부를 (dx, dy, revealed, state)로 인코딩한 124-dim 관측을 SAC에 직접 입력.  
*가설*: 정보가 많을수록 정책이 더 풍부하게 학습할 것이다.  
*결과*: 0% 커버리지. 에이전트가 시작 위치 근처에서 진동하다 max_steps 도달.  
*교훈*: 희박 보상 환경에서 고차원 관측은 학습 신호 대 잡음비를 악화시킨다.

**v2 — Potential-based shaping 추가 (커버리지 6%)**

*시도*: 학습 신호를 조밀하게 만들기 위해 가장 가까운 미처리 작물까지의 거리를 잠재 함수로 사용하는 reward shaping 추가.  
*가설*: 작물에 가까워질수록 보상이 누적되면 에이전트가 자연스럽게 작물을 향해 이동할 것이다.  
*결과*: 약간 개선되었으나(6%) 에이전트가 한 작물 근처에서 진동하며 보상을 누적하는 **정착 함정(reward hacking)**에 빠졌다.  
*교훈*: 단순 근접 보상은 부작용을 만든다. Ng et al.(1999)의 잠재 함수 차분 형태(F = γφ(s') − φ(s))로 교체하여 최적 정책 불변성을 보장.

**v3 — nav_flags 추가 (커버리지 56%)**

*시도*: 관측에 4방향(N/S/E/W) 이동 가능 여부를 명시한 nav_flags(4-dim) 추가.  
*가설*: 에이전트가 어느 방향이 막혀 있는지를 알면 충돌 시도를 줄일 수 있을 것이다.  
*결과*: 6%에서 56%로 대폭 향상. 충돌 반복 패턴이 사라졌다.  
*교훈*: 연속 환경에서는 "충돌 패널티" 같은 사후 신호보다 "이동 가능 여부" 같은 사전 정보가 훨씬 효율적이다.

**v4 — 관측 단순화(28-dim) + Level 2 고정 시작 (커버리지 96.7%)**

*시도*: 124-dim → 28-dim으로 축소하고(top-5 nearest crops만), 모든 에피소드를 Level 2 헤드랜드에서 시작하도록 고정.
*가설*: 정책이 다뤄야 할 정보를 가장 중요한 것에 집중시키면 전체 환경에서도 안정적으로 수렴할 것이다.
*결과*: 96.7% 성공·99.9% 커버리지에 도달.  
*교훈*: 관측의 양보다 질이 중요하다. 같은 정보량이라도 정책 결정에 직결되는 정보로 집약하면 학습 효율이 비약적으로 향상된다.

### 5.4 연속 계층형 RL 실험 (실패)

**시도**: 이산 환경에서 성공한 계층 구조를 연속 환경에 그대로 이식하였다. ContinuousLaneExecutorEnv(목표 레인 지정 가능한 SAC LL)와 HighLevelContinuousEnv(DQN HL)를 구성하고, SAC·PPO·A2C 세 LL 알고리즘을 각각 1M 스텝 학습.

**가설**: 이산 환경에서 계층 분리가 성공한 이상, 같은 구조를 연속 환경에 적용해도 잘 작동할 것이다.

**결과**: 세 알고리즘 모두 부분적으로만 수렴.

| LL 알고리즘 | 커버리지 |
|------------|---------|
| SAC | 41.7% |
| PPO | 21.7% |
| A2C | 16.7% |

추가로 VecNormalize 불일치 버그를 발견하였다. SAC LL을 VecNormalize로 학습시키면, HL 환경이 raw obs를 직접 전달할 때 분포가 달라 정책이 망가졌다. VecNormalize를 제거해 해결.

**교훈**: 이산 LL과 달리 연속 LL은 "지정 레인으로 이동 후 그 레인만 처리하고 종료"하는 **goal-conditioned 태스크**다. 현재 1M 스텝 설정에서는 낮은 커버리지에 머물렀으며, 더 긴 학습 또는 Hindsight Experience Replay 같은 goal-conditioned 전용 기법의 검토가 필요하다.

### 5.5 학습 인프라 개선 — SAC 학습 속도

**시도**: 초기 SAC 학습은 73 fps에 불과해 1M 스텝에 약 4시간이 걸렸다. CPU 병목으로 GPU(RTX 3080)가 거의 활용되지 않았다.

**가설**: n_envs를 늘리고 train_freq를 키워 환경 수집과 그라디언트 업데이트의 비율을 조정하면 throughput이 개선될 것이다.

**결과**: 개발 중 측정 기록에서는 n_envs=4, train_freq=16, batch_size=4,096 구성에서 약 635 fps를 기록하였다. 원시 벤치마크 로그가 현재 저장소에 없어 정밀한 재검증에는 한계가 있다.

**교훈**: 해당 개발 환경에서는 업데이트 빈도와 병렬 수집 설정이 처리량에 큰 영향을 주었다. 하드웨어와 환경에 따라 결과가 달라질 수 있으므로 동일 조건의 반복 벤치마크가 필요하다.

### 5.6 코드 버그 발견 및 수정

리뷰 과정에서 발견·수정한 주요 버그.

| 버그 | 영향 | 수정 |
|------|------|------|
| `_goal_reached` 레인 전환 시 미리셋 | HL이 다음 레인을 지정할 때 LL의 goal-reaching 보너스(+2.0)가 누락 → 두 번째 이후 레인에서 학습 신호 약화 | HL.step()에서 매 전환마다 False로 리셋 |
| `_prev_potential` `__init__`에서 미초기화 | reset() 전에 step() 호출 시 AttributeError 발생 가능 | `__init__`에서 0.0으로 초기화 |
| `_handle_move` bounds check 누락 | 작은 맵 변형 시 IndexError 가능성 | 0 ≤ nr < H and 0 ≤ nc < W 검사 추가 |

이 중 `_goal_reached` 버그는 §5.4의 연속 계층형 학습 부진의 잠재적 원인 중 하나로 추정된다. 수정 후 재학습으로 검증할 필요가 있다.
