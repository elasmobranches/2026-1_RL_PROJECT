# 강화학습 프로젝트 보고서
## 정밀농업 자율 로봇을 위한 커스텀 강화학습 환경 설계 및 적용

**학과**: AI로봇학과 | **제출일**: 2026-06-15 | **사용 언어**: Python 3.10

---

## 1. 문제 정의

### 1.1 왜 이 문제인가

스마트팜 관련 연구를 하다 보면 실제로 가장 귀찮은 게 작물 상태 확인이다. 드론이나 카메라로 전체를 찍는 방법도 있지만, 해충이 생긴 건 가까이서 봐야 알 수 있고 수확 적기도 직접 확인해야 정확하다. 자율 이동 로봇이 직접 돌아다니면서 필드를 순찰하고 조치까지 취하는 시스템을 강화학습으로 구현해보기로 했다.

### 1.2 왜 강화학습인가

이 문제는 세 가지 이유에서 강화학습이 적합하다.

**순차적 의사결정**: 현재의 이동 선택이 이후 도달 가능한 영역과 잔여 작업량을 바꾼다. 매 스텝의 행동이 미래 보상에 영향을 주므로 단순 분류·회귀가 아닌 누적 보상 최대화 문제다.

**부분 관측(POMDP)**: 로봇은 예찰(Scout) 행동을 하기 전에는 작물의 실제 상태를 알 수 없다. "정보를 얻는 행동(예찰)"과 "정보를 활용하는 행동(조치)"이 분리되어 있어, 에이전트(또는 계층적 구조에서는 상위 정책)가 탐색과 활용의 순서를 스스로 결정해야 한다.

**경로 최적화의 조합론적 복잡성**: 5개 레인 방문 순서만 해도 최소 120가지(5!) 이상이며, 매 에피소드마다 작물 상태가 랜덤하게 바뀌므로 고정 규칙으로는 최적 대응이 불가능하다. "수확 대기 발견 → 수확" 같은 개별 행동은 규칙화할 수 있어도, 이동 경로는 보상 신호로 학습하는 게 적합하다.

### 1.3 해결 목표

| 조건 | 설명 |
|------|------|
| 충돌 없는 이동 | 작물 구역·벽과 충돌하지 않고 주행 레인만 이동 |
| 전체 예찰 | 모든 작물 셀을 빠짐없이 스캔 |
| 상태 기반 조치 | 예찰 결과에 따라 정상 확인/수확/방제 구분 후 조치 |

### 1.4 접근 방법 개요

문제를 단계적으로 발전시켰다. 각 단계는 이전 단계의 한계를 개선하는 방식으로 진행됐다.

| 단계 | 핵심 아이디어 | 최고 성능 |
|------|-------------|---------|
| Step 1 | 이산 격자 + MaskablePPO | 96% |
| Step 2 → 3 | 계층 구조 + 거리 인식 관측 | **100%** |
| Step 4 | 연속 공간 + SAC | 96.7% |

### 1.5 ROS2 아키텍처 (배포 관점)

시뮬레이션에서는 Gymnasium 환경이 5개 ROS2 노드를 통합 구현한다.

```
[SceneObserverNode] → [RLAgentNode] → [ActionExecutorNode]
[EpisodeManagerNode] ← [RewardCalculatorNode] ←──────────┘
```

## 2. Step 1: 이산 Flat RL

### 2.1 환경 설계

**왜 이 환경인가**: 문제를 가장 단순하게 표현하는 것부터 시작했다. 이산 격자에서 칸 단위로 이동하며 예찰·조치를 수행하는 구조다.

현실 온실의 수직재배 구조를 모델링한 **2열 재배단** 격자 맵을 사용한다. 토마토·파프리카 같은 작물은 줄기 양쪽으로 열매가 분포하므로, 한쪽 레인에서 접근하면 안쪽 열만 스캔 가능하고 바깥쪽 열은 맞은편 레인에서만 접근 가능하다. 이 **양면 접근 제약**이 5개 레인을 모두 방문해야 하는 이유다.

![맵 구조](map_structure.png)

```
최종 맵 (n_beds=4, field_height=8): H=12, W=15

r0:  W  W  W  W  W  W  W  W  W  W  W  W  W  W  W
r1:  W  P  P  P  P  P  P  P  P  P  P  P  P  P  W   ← 헤드랜드
r2:  W  P  C  C  P  C  C  P  C  C  P  C  C  P  W   ← 필드 행 (r2~r9)
r10: W  P  P  P  P  P  P  P  P  P  P  P  P  P  W   ← 헤드랜드
r11: W  W  W  W  W  W  W  W  W  W  W  W  W  W  W

P=주행 레인, C=작물 셀, W=벽 / 레인: col 1,4,7,10,13 (5개)
```

헤드랜드는 레인 간 전환의 유일한 통로다.

**작물 상태 동역학 (POMDP)**: 각 작물은 에피소드 시작 시 아래 분포로 초기화되지만 에이전트에게는 숨겨져 있다. Scout 액션을 수행해야만 실제 상태가 공개된다. 이것이 §1.2의 POMDP 특성이다.

| 초기 상태 | 확률 |
|----------|------|
| 정상 | 60% |
| 수확 필요 | 25% |
| 방제 필요 | 15% |

**관측 공간**: `Box(0,1, shape=(720,))` — 4채널(맵구조/에이전트위치/예찰마스크/예찰결과) × 12 × 15

**행동 공간**: `Discrete(7)` + **Action Masking**

| 인덱스 | 액션 | 마스킹 조건 |
|--------|------|------------|
| 0~3 | 이동 (상/하/좌/우) | 목적지가 작물/벽 |
| 4 | 예찰 (Scout) | 인접 미예찰 셀 없음 |
| 5 | 수확 | 인접 수확대기 셀 없음 |
| 6 | 방제 | 인접 방제대기 셀 없음 |

Action Masking은 무효 액션의 로짓을 −∞로 치환해 선택 확률을 0으로 만든다.

**보상**: 매스텝 −0.1 / 충돌 −2.0 / 예찰 +1.0 / 정상확인 +0.5 / 수확 +10.0 / 방제 +8.0 / 전체완료 +20.0

**종료**: 전체 작물 처리 완료 OR max_steps=540 초과

### 2.2 알고리즘: MaskablePPO

이산 행동 공간 + 동적 Action Masking이 동시에 필요하다. sb3-contrib의 MaskablePPO가 두 조건을 모두 지원한다. 16개 병렬 환경으로 빠른 수집 후 배치 업데이트.

| 파라미터 | 값 |
|--|--|
| n_envs | 16 |
| total_timesteps | 1,500,000 |
| learning_rate | 3×10⁻⁴ |

### 2.3 결과

| 성공률 | 커버리지 | 평균 스텝 |
|--------|---------|---------|
| 96% | 99.8% | 147~211 |

에이전트는 명시적 경로 알고리즘 없이 보상 설계만으로 **Boustrophedon(지그재그) 패턴**을 자발적으로 학습했다.

*(agent_demo.gif 참조)*

### 2.4 한계

평균 스텝 210 vs 이론 최단 ~104 — 약 40% 비효율. 전역 레인 방문 순서를 최적화하지 못했다. "어느 레인을 다음에 갈지"(전역 계획)와 "지금 이 레인에서 어떻게 움직일지"(국소 실행)가 하나의 정책에 혼재되어 있기 때문이다.

## 3. Step 2/3: 이산 계층형 RL

### 3.1 왜 계층 구조인가

Step 1의 Flat RL은 전역 계획과 국소 실행을 동시에 학습해야 한다. 이 두 문제를 분리해 각각에 특화된 정책을 학습하면 더 효율적일 것이라 판단했다.

```
High-level (레인 선택) → 목표 레인 지정 → Low-level (레인 처리)
```

### 3.2 환경 설계 변경

Step 1의 FarmEnv를 두 계층으로 분리했다.

**Low-level (LaneExecutorEnv)**: FarmEnv 확장. ch4(목표 레인 마스크) 추가 → 관측 5채널 × 900-dim. High-level이 지정한 목표 레인을 Low-level에 전달하는 채널이다. 목표 레인의 작물이 모두 처리되면 에피소드 종료 (max_steps=180/레인).

**High-level (HighLevelFarmEnv)**: Low-level 정책을 실행기로 사용. obs = [레인별 완료율(5), 현재 위치→각 레인 거리(5)] = 10-dim. 거리 정보를 포함시킨 것이 핵심 — 이를 통해 "가장 가까운 미완료 레인 선택" 전략을 자동으로 학습할 수 있다.

### 3.3 Step 2 → Step 3: 거리 인식 관측 추가

**Step 2**: High-level obs에 완료율만 포함 → 레인 거리 정보 없음 → 비효율적 순서 학습.

**Step 3**: High-level obs에 **거리 정보 추가** → DQN이 Greedy nearest-lane 전략 자동 발견.

확인 실험: Step 3 DQN을 "항상 가장 가까운 레인 선택"하는 Greedy 규칙과 비교했더니 **결과가 완전히 동일** (141±3 steps, 2.0 visits). 올바른 관측이 주어지면 RL이 최적 휴리스틱을 스스로 발견한다.

*(results_greedy_vs_rl.png 참조)*

### 3.4 알고리즘

| | Low-level | High-level |
|--|--|--|
| 알고리즘 | MaskablePPO | DQN (Step 3) |
| 학습 스텝 | 700k | 30k |
| 역할 | 레인 내 이동·예찰·조치 | 레인 방문 순서 결정 |

Low-level도 Action Masking 적용. High-level은 5개 레인 선택 문제(Discrete 5)라 DQN이 충분했다.

### 3.5 결과

| | Step 2 | Step 3 |
|--|--|--|
| 성공률 | 98% | **100%** |
| 커버리지 | 99.8% | **100%** |
| 평균 스텝 | 198 | **141 ± 3** |
| 레인 방문 | 2.3회 | **2.0회** |

Step 3이 이 프로젝트의 최고 성능. 2.0회 방문으로 5개 레인을 100% 커버했다 — Low-level이 목표 레인으로 이동하는 경로에서 다른 레인의 작물도 자연스럽게 처리하는 **암묵적 다중 레인 커버리지** 효과 덕분이다.

*(agent_demo_step3.gif 참조)*

## 4. Step 4: 연속 환경 + SAC

### 4.1 왜 연속 환경인가

이산 격자는 로봇이 칸 단위로 순간이동하는 단순화다. 실제 로봇은 연속 공간에서 속도 명령으로 움직인다. 이산 환경에서 검증한 정책 설계 원칙이 연속 제어로 확장될 수 있는지 검증하고자 했다.

### 4.2 환경 설계

이산 환경의 격자 구조(2열 재배단, 5개 레인, 헤드랜드)를 그대로 유지하되, 공간과 이동을 연속으로 바꿨다.

| | 이산 환경 | 연속 환경 |
|--|--|--|
| 위치 | 격자 인덱스 | 실수 좌표 (x, y) |
| 이동 | 칸 단위 (7가지 액션) | 속도 벡터 (vx, vy) |
| 예찰/조치 | 명시적 Scout 액션 | 거리 임계값 이내 자동 |

**관측 공간**: `Box(-1,1, shape=(28,))`

| 구성 | 차원 |
|------|------|
| robot_pos + heading | 4 |
| nav_flags (4방향 이동 가능 여부) | 4 |
| 가장 가까운 미처리 작물 5개 (dx,dy,rev,state) | 20 |

nav_flags가 핵심 — 로봇이 어느 방향이 막혀 있는지 명시적으로 알게 해 연속 환경의 충돌 반복 문제를 해결했다. (§6.3 참조)

**보상**: 이산 대비 스케일 조정 (R_STEP=−0.05, R_COLLISION=−1.0) + Potential-based shaping:

`F = γ·φ(s') − φ(s)`,  `φ(s) = −min_dist × 0.3`

(Ng et al. 1999 — 최적 정책 불변성 보장하면서 학습 신호 조밀화)

**커리큘럼**:
- Level 0: 작물 바로 옆 스폰 (이동·예찰 먼저 학습)
- Level 2: 헤드랜드 정상 스폰 (전체 에피소드)

### 4.3 알고리즘: SAC

연속 행동 공간 → 연속 제어 알고리즘 필요. SAC의 **최대 엔트로피 프레임워크**를 선택한 이유: 희박 보상 환경에서 자동으로 탐색 정도를 조절한다. 동일 환경에서 TD3·TQC와 비교했을 때 SAC가 압도적으로 우수했다. (§5.2 참조)

| 파라미터 | 값 |
|--|--|
| total_timesteps | 3,000,000 |
| train_freq | 16 |
| batch_size | 4,096 |

### 4.4 결과

| 성공률 | 커버리지 | 평균 스텝 |
|--------|---------|---------|
| 96.7% | 99.9% | 157 |

*(agent_demo_sac_curriculum.gif 참조)*

## 7. R&D 시행착오

### 7.1 1열 → 2열 재배단

초기 1열 재배단(24셀)에서 2열(64셀)로 변경. 현실 구조 반영.

### 7.2 스텝 패널티 강화 실험 (실패)

−0.1→−0.3: 성공률 96%→80% 급락. 레인 완료 실패율 증가. "보상보다 관측"이라는 교훈.

### 7.3 SAC 연속 환경 반복 개선

| 시도 | 변경 | 결과 | 원인 |
|------|------|------|------|
| v1 | 기본 SAC, 124-dim obs | 0% | 희박 보상 + 고차원 |
| v2 | Potential shaping | 6% | 정착 함정 |
| v3 | **nav_flags 추가** | 56% | 방향 인식 해결 |
| **v4** | **obs 단순화 + 커리큘럼** | **96.7%** | — |

v2→v3 핵심: nav_flags 없이는 로봇이 작물 구역이 막혀 있다는 걸 obs로 알 방법이 없어 계속 충돌 시도.

### 7.4 연속 계층형 RL 실험 (실패)

SAC/PPO/A2C LL + DQN HL 조합을 시도했다.

| LL | 커버리지 | 원인 |
|----|---------|------|
| SAC | 41.7% | goal-conditioned 학습 1M 부족 |
| PPO | 21.7% | 동일 |
| A2C | 16.7% | 동일 + 불안정 |

이산 LL과 달리 연속 LL은 "지정 레인으로 이동 후 그 레인만 처리"라는 목표 조건부(goal-conditioned) 태스크 — 3M+ 스텝 또는 별도 커리큘럼이 필요하다. 추가로 VecNormalize 불일치 버그(HL이 raw obs 전달 → 분포 불일치)도 발견해 수정했다.

### 7.5 코드 버그 수정

| 버그 | 수정 |
|------|------|
| `_goal_reached` 레인 전환 시 미리셋 | 전환마다 False 리셋 |
| `_prev_potential` 미초기화 | 0.0으로 초기화 |
| `_handle_move` bounds check 없음 | 경계 검사 추가 |

## 8. 결론

### 8.1 요약

| 단계 | 핵심 기여 |
|------|----------|
| Step 1 | Action Masking 필수성 확인, Boustrophedon 자발 학습 |
| Step 2→3 | 계층 분리 + 거리 인식으로 100% 달성, RL이 Greedy 전략 자동 발견 |
| Step 4 | 연속 공간 확장, SAC 엔트로피 탐색의 필요성 확인 |

### 8.2 핵심 교훈

보상 함수 튜닝보다 **관측 공간 설계**가 결정적이었다. nav_flags 추가 하나로 연속 환경 커버리지가 6%→56%로 뛰었고, 거리 정보 추가로 Hierarchical RL이 90%→100%가 됐다.

### 8.3 향후 연구

- 다중 로봇 협력(MARL): MAPPO 기반 분할 예찰
- Hierarchical SAC: 3M+ 스텝으로 재시도
- 실제 ROS2 배포: `cmd_vel` 토픽 직접 연결
